# 虚拟机分类
## 按Hypervisor 架构（部署方式）
根据虚拟化层（VMM/Hypervisor）是否直接运行在硬件上，分为 **Type 1（裸金属）** 和 **Type 2（寄居 / 桌面）** 两大类。
#### 1. Type 1 Hypervisor（裸金属虚拟化 / Bare-Metal）
- **原理**：Hypervisor 直接安装在物理硬件上，**不依赖宿主 OS**，全权管理 CPU、内存、I/O 等资源。
- **性能**：极高，损耗仅 **5%–10%**，接近物理机。
- **稳定性**：极高，Hypervisor 极小、独立，宿主 OS 故障不影响。
- **典型产品**：
    - VMware **ESXi / vSphere**（企业主流）
    - Microsoft **Hyper-V Server**（Windows 专属）
    - **KVM**（Linux 内核原生，开源主流）
    - **Proxmox VE**（KVM + 容器，带 Web 管理）
    - Citrix **XenServer** / **XCP-ng**（开源）
#### 2. Type 2 Hypervisor（寄居虚拟化 / Hosted）
- **原理**：先装 Windows/Linux/macOS，再装虚拟机软件（作为应用程序）。
- **性能**：中等，损耗 **10%–30%**（资源需经宿主 OS 二次转发）。
- **易用性**：极高，安装简单、操作友好。
- **典型产品**：
    - VMware **Workstation Pro / Player**（Windows/Linux）
    - Oracle **VirtualBox**（免费开源、跨平台）
    - VMware **Fusion**（macOS）
    - Parallels **Desktop**（macOS 运行 Windows）
## 按**虚拟化实现方式**
#### 1. 全虚拟化（Full Virtualization）

- **特点**：**完整模拟整套硬件**（CPU、网卡、磁盘）。客户机 OS **完全不知情、无需修改**。
- **优点**：兼容性极强（几乎支持所有 OS）。
- **缺点**：纯软件模拟，**性能开销大**。
- **代表**：早期 VMware、QEMU、VirtualBox（纯软件模式）。

#### 2. 半虚拟化（Paravirtualization）

- **特点**：不模拟硬件，**修改客户机内核**，通过 `Hypercall` 直接与 Hypervisor 通信。
- **优点**：**性能接近原生**，开销极低。
- **缺点**：必须用**修改版 OS**（如专用 Linux 内核），Windows 不支持。
- **代表**：Xen（PV 模式）、早期 UML。

#### 3. 硬件辅助虚拟化（Hardware-Assisted）

- **特点**：依赖 CPU 虚拟化指令集（**Intel VT-x / AMD-V**），由硬件加速指令翻译。
- **优点**：**性能最高、最安全**，OS 无需修改。
- **现状**：**现代主流技术**，Type 1 几乎都用此模式。
- **代表**：KVM、Hyper-V、VMware ESXi、Xen（HVM 模式）。

#### 4. 操作系统级虚拟化（容器，非传统 VM）

- **特点**：**共享宿主机内核**，仅隔离进程、文件、网络（不是完整虚拟机）。
- **代表**：Docker、LXC、Podman。
# Hyper-V
**Hyper-V 是微软官方自带的硬件虚拟化技术（Type‑1 裸金属虚拟化）**，相当于 Windows 内置的 “虚拟机管理系统”，用来在 Windows 上跑虚拟机、Linux、甚至其他 Windows 系统。
---
## 1. 它属于哪一类虚拟化？

严格来说：

- **Hyper-V 本身是 Type‑1 裸金属 Hypervisor**
- 但它和 Windows 内核深度绑定，开机时先启动 Hyper‑V 底层，再把 Windows 本身跑成一个 “特殊虚拟机”

所以它**性能非常强**，远好于 VMware Workstation、VirtualBox 这类普通 Type‑2 软件。

---

## 2. 它能干什么

- 安装并运行 **Linux、Windows、其他系统** 的完整虚拟机
- 搭建测试环境、服务器环境
- **WSL2 底层就是靠 Hyper‑V 实现的**
- 企业里用来做服务器虚拟化、云桌面
