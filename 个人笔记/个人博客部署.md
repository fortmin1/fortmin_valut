# 个人博客部署笔记

## 一、静态站点生成器（SSG）

### 主流工具对比

| 工具             | 技术栈      | 适合场景        | 构建速度   | 特点                              |
| -------------- | -------- | ----------- | ------ | ------------------------------- |
| **Astro**      | JS（框架无关） | 博客、文档、营销站   | 快      | 市场份额第一（~25%），零 JS 输出，Islands 架构 |
| **Hugo**       | Go       | 大型博客、文档站    | 极快（秒级） | 适合 1000+ 页面的大站点                 |
| **Hexo**       | Node.js  | 中文个人博客      | 中等     | 中文生态最好，GitHub Pages 一键部署        |
| **VuePress**   | Vue 2    | 技术文档        | 较慢     | 已被 VitePress 取代                 |
| **VitePress**  | Vue 3    | 技术文档        | 极快     | Vue 官方推荐，毫秒级热更新                 |
| **Next.js**    | React    | 全栈应用 + 静态导出 | 中等     | 最全面，支持 SSR/ISR                  |
| **Docusaurus** | React    | 开源项目文档      | 中等     | Meta 官方维护                       |
| **Eleventy**   | JS       | 极简博客        | 快      | 灵活，无框架绑定                        |
| **Jekyll**     | Ruby     | 简单博客        | 中等     | GitHub Pages 原生支持               |

### VuePress / VitePress 的能力边界

**擅长的：**
- 技术文档站、个人博客
- Markdown 编写体验优秀
- Vue 组件原生嵌入
- 中文生态丰富

**做不到的：**
- **CMS 功能** — 没有后台管理界面、用户权限、审核流程，非技术人员无法编辑内容
- **动态服务端逻辑** — 无法处理用户登录、数据库查询、实时推荐
- **大规模内容管理** — 内容通过 Git 管理，没有角色权限系统
- **实时交互** — 无法做实时搜索、个性化推荐、在线聊天
- **电商/复杂应用** — 不适合购物车、支付等需要服务端状态的功能

> 核心定位：Markdown → 静态 HTML 的文档/博客生成器，适合内容驱动、更新频率不高的站点。

## 二、CMS 与动态应用

### CMS（内容管理系统）

给非技术人员用的可视化内容编辑后台，如 WordPress、Ghost、Strapi。

```
静态站（如 VuePress）：写 Markdown → git commit → push → 构建
CMS：浏览器登录后台 → 可视化编辑 → 点"发布" → 上线（不需要懂代码）
```

### 动态应用

服务器根据请求实时生成不同内容，如淘宝（千人千面）、微博（实时时间线）。

需要的能力：用户登录、数据库、实时交互、个性化推荐、支付交易 — 这些都是静态站点做不到的。

## 三、部署方式

### 部署平台对比

| 平台                   | 免费额度     | 国内访问   | 构建方式               | 自定义域名 | 适合场景       |
| -------------------- | -------- | ------ | ------------------ | ----- | ---------- |
| **GitHub Pages**     | 无限（公开仓库） | 慢/不稳定  | Actions / gh-pages | 支持    | 文档站、个人博客   |
| **Vercel**           | 100GB/月  | 一般     | 自动检测框架             | 支持    | Next.js 项目 |
| **Netlify**          | 100GB/月  | 一般     | 自动构建部署             | 支持    | 广泛框架支持     |
| **Cloudflare Pages** | 无限带宽     | **较快** | Git 集成             | 支持    | 高流量、国内用户   |

### 典型部署流程

```
方案 A：GitHub Pages + gh-pages（传统方式）
  push → CI 触发 → build → force push gh-pages 分支 → 自动发布

方案 B：GitHub Pages + GitHub Actions（推荐）
  push → Actions 触发 → build → actions/deploy-pages → 自动发布

方案 C：Vercel / Netlify（零配置）
  push → 平台自动检测 → build → 自动部署到全球 CDN

方案 D：Cloudflare Pages
  push → Cloudflare 自动构建 → Workers + Pages → 边缘计算 + 静态托管
```

### Hexo 部署到 GitHub Pages

最经典的组合，Hexo 内置支持：

```bash
npm install hexo-deployer-git --save
```

```yaml
# _config.yml
deploy:
  type: git
  repo: https://github.com/username/username.github.io.git
  branch: main
```

```bash
hexo clean && hexo generate && hexo deploy
```

## 四、在 VitePress 博客中加音乐播放器

音乐播放是浏览器端功能（HTML5 Audio API），静态站点完全可以实现。

### 方案 A：APlayer 组件

```bash
npm install aplayer
```

```vue
<!-- .vuepress/components/MusicPlayer.vue -->
<template>
  <div ref="player" />
</template>

<script setup>
import { ref, onMounted } from 'vue'
import APlayer from 'aplayer'
import 'aplayer/dist/APlayer.min.css'

const player = ref(null)

onMounted(() => {
  new APlayer({
    container: player.value,
    audio: [{
      name: '歌曲名',
      artist: '歌手',
      url: 'https://example.com/song.mp3',
      cover: 'https://example.com/cover.jpg'
    }]
  })
})
</script>
```

### 方案 B：iframe 嵌入外部播放器

最省事，直接嵌入网易云等平台：

```html
<iframe
  frameborder="no" border="0"
  width="330" height="86"
  src="//music.163.com/outchain/player?type=2&id=歌曲ID&auto=0&height=66"
/>
```

### 注意事项

- 音频文件**不要放 GitHub 仓库**（仓库大小限制），应托管到 OSS/CDN（阿里云 OSS、Cloudflare R2）
- 公开站点放版权音乐有法律风险，建议用免版权音乐或嵌入平台播放器

## 五、选型建议

| 需求 | 推荐 |
|---|---|
| 写技术文档 | VitePress 或 Docusaurus |
| 写中文个人博客 | Hexo 或 Hugo |
| 追求极致性能 | Astro |
| 内容量大（1000+ 页） | Hugo |
| 需要动态交互 | Next.js 或 Nuxt |
| 国内用户访问 | Cloudflare Pages 部署 |
